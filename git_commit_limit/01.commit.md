# commit 信息：`<type>(<scope>): <subject>  # 50 字内，句尾无点`
## 1.type信息
**type** 必为下列单选  
| 代号 | 含义 |
|---|---|
| feat | 新功能 |
| fix | 修 Bug |
| docs | 仅文档 |
| style | 格式、补分号等无逻辑改动 |
| refactor | 重构（不是 feat 也不是 fix） |
| test | 测试相关 |
| chore | 杂务（升级依赖、改构建脚本等） |

**scope** 可选，
- scope 就是“你这次改的是哪一块”，用文件夹名、模块名、功能域都行，怎么直观怎么来
- 用模块/文件夹名即可，保持小写单词  
- 例：`parser`、`resolver`、`cli`、`ast`, 

**subject** 用现在时态动词开头，首字母小写，**不加句号**

## 3.可选页脚（footer）
| 关键字                            | 作用                |
| ------------------------------ | ----------------- |
| `BREAKING CHANGE: <描述>`        | 声明不兼容变更 → 自动升主版本号 |
| `Closes #123` / `Fixes #123`   | 关闭对应 Issue/PR     |
| `Co-authored-by: name <email>` | 多作者               |
